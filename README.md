# peduha2.0

Сайт Володимирського педагогічного фахового коледжу на Flask з адмінпанеллю, SQLite-базою, системою статей, відгуками випускників, формами для вступників і єдиною структурою фронтенду.

## Що є в проєкті

- публічний сайт з головною, сторінками розділів, статтями та сторінкою вступу
- адмінпанель для керування статтями, меню, промоблоками, користувачами, заявками й відгуками випускників
- SQLite як основна база даних
- завантаження зображень і документів через адмінку
- окремі сторінки для договору оферти, курсів, запитань вступників і відгуків випускників

## Технології

- Python
- Flask
- SQLite (`sqlite3`)
- Jinja2
- HTML / CSS / JavaScript

## Структура проєкту

- `app.py` — файл запуску застосунку
- `app/` — основна логіка Flask-застосунку
  - `app/__init__.py` — пакет застосунку
  - `app/core.py` — конфіг, спільні хелпери, фільтри, auth-декоратори
  - `app/public_routes.py` — публічні маршрути сайту
  - `app/admin_routes.py` — маршрути адмінпанелі
- `data/`
  - `data/db.py` — робота з БД, схема, ініціалізація, стартові дані
  - `data/app.db` — локальна SQLite-база
- `templates/` — усі Jinja-шаблони сайту та адмінки
- `static/`
  - `static/css/styles.css` — стилі
  - `static/js/main.js` — клієнтський JavaScript
  - `static/images/` — статичні зображення
  - `static/uploads/` — файли, завантажені через адмінку
- `pages/` — HTML-вміст для сторінок `/page/<slug>`

## Як запустити локально

1. Створи та активуй віртуальне середовище
2. Встанови залежності:

```bash
pip install -r requirements.txt
```

3. Запусти сайт:

```bash
python app.py
```

Після першого старту автоматично:
- створиться `data/app.db`, якщо її ще немає
- створяться потрібні таблиці
- згенерується owner-користувач, якщо база порожня

## Вхід в адмінку

Сторінка входу:
- `/login`

Якщо база створюється вперше, owner-користувач генерується автоматично в `data/db.py`.
Логін і згенерований пароль виводяться в консоль під час ініціалізації порожньої бази.

## Основні можливості адмінки

- статті:
  - створення, редагування, видалення
  - категорії, дати, розділи
  - завантаження головного фото, галереї та документів
- меню сайту:
  - створення ієрархії розділів
  - редагування назв, URL і порядку
- промоблоки головної сторінки
- заявки на підготовчі курси
- запитання вступників
- відгуки випускників:
  - короткі картки на головній
  - окрема сторінка списку
  - детальна сторінка кожного відгуку
- перегляд таблиць бази даних у режимі read-only
- керування користувачами

## Дані та файли

- усі завантажені зображення й документи зберігаються в `static/uploads/`
- основна база лежить у `data/app.db`
- логіка схеми та міграцій — у `data/db.py`
